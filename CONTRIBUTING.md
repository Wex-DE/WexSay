# Contributing to WexSay

First off, thank you for checking out WexSay! We appreciate your interest in making this platform better.

Please note that **the core source code of the WexSay Android application and Backend API is proprietary and private.** While you cannot submit code modifications to the core application directly, there are many ways you can contribute to the community and help shape the project!

---

## How You Can Contribute

### 1. Report Bugs & Issues
* Explore the app and look for bugs, crashes, layout issues, calling latency, or notification delays.
* Open an issue using the [Bug Report Template](.github/ISSUE_TEMPLATE/bug_report.md). Provide details, logs, and screenshots.

### 2. Suggest Features & Enhancements
* Have ideas on how to improve WexSay's features or UI/UX?
* Submit feature requests using the [Feature Request Template](.github/ISSUE_TEMPLATE/feature_request.md).

### 3. Help Other Testers
* Engage in [GitHub Discussions](https://github.com/Wex-DE/WexSay/discussions).
* Help answer questions, discuss UI styling preferences, or share device compatibility results.

### 4. Improve Documentation
* Spot a typo in our guides or want to improve a readme description?
* Submit a Pull Request (PR) to modify files in this public repository!

---

## Bug Report Guidelines

To help our engineering team fix bugs quickly, please include:
* **Device details:** Brand, model, and Android OS version.
* **Reproduction steps:** Exactly what steps you took to trigger the bug.
* **Observed vs. Expected behavior:** What happened and what should have happened.
* **Logs / Screenshots:** Any relevant files.

---

## Community Code of Conduct
By participating in this project, you agree to abide by our [Code of Conduct](CODE_OF_CONDUCT.md). Please be respectful and professional in all communications.
